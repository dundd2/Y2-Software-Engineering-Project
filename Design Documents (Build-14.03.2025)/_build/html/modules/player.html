Player Module

This module contains the Player class that represents a player in the Property Tycoon game, managing player attributes, movement, and interactions.

Key Features

  • Player initialization and configuration: Creates and configures player objects

  • Animation system: Provides smooth animations for player movement and visual effects

  • Token visualization: Manages player token appearance and customization

  • Property management: Tracks owned properties and transactions

  • Financial operations: Handles player money and financial transactions

  • Jail state management: Handles jail status, jail cards, and jail turns

  • Bankruptcy processing: Manages the bankruptcy state and asset liquidation

  • Voluntary exit handling: Processes players who choose to leave the game early

  • Asset calculation: Determines player’s total assets including properties

  • AI player integration: Special handling for AI players

  • Visual feedback: Provides visual cues for player state and actions

Player Attributes

Each Player object maintains several key attributes:

  • name: Player’s display name

  • player_number: Numeric identifier (1-6) that also determines token appearance

  • is_ai: Boolean indicating if the player is AI-controlled

  • money: Current amount of money the player has

  • properties: List of properties owned by the player

  • position: Current position on the board (1-40)

  • in_jail: Boolean indicating if the player is in jail

  • jail_turns: Number of turns spent in jail

  • jail_cards: Get Out of Jail Free cards held

  • stay_in_jail: Boolean indicating if player chooses to stay in jail

  • bankrupt: Boolean indicating if player is bankrupt

  • voluntary_exit: Boolean indicating if player has voluntarily left the game

  • final_assets: Total value of assets at time of exit

Animation System

The Player class features an advanced animation system:

  • Movement animation: Smooth transitions between board spaces

  • Path-based movement: Animates along generated paths for dice results

  • Bounce effect: Special animation for winner celebration

  • Active state highlighting: Visual emphasis for the current player

  • Jail visual state: Special appearance when in jail

  • Voluntary exit visual state: Faded appearance when player has left the game

  • Animation timing: Speed and progress controls for movement

  • Glow effects: Visual enhancement for highlighted states

Player Token System

The Player class provides a customizable token system:

  • Image loading: Loads token images from asset files

  • Fallback generation: Creates placeholder tokens if images can’t be loaded

  • Color customization: Each player has a distinct color

  • Size adaptation: Resizes tokens based on screen resolution

  • Visual effects: Adds glow and shadow effects to tokens

  • Player number display: Shows player number on tokens

  • State visualization: Changes token appearance based on player state

Financial System

The Player class manages financial operations:

  • Money tracking: Maintains the current money amount for the player

  • Payment handling: Methods for paying money to other players or the bank

  • Receipt handling: Methods for receiving money from other players or the bank

  • Affordability checking: Determines if player can afford a transaction

  • Purchase processing: Handles buying properties and houses/hotels

  • Asset calculation: Determines total player assets including property values

  • Money display: Formats money amounts for display

Property Management

The Player class provides comprehensive property management:

  • Property ownership: Tracks which properties the player owns

  • Property acquisition: Methods for buying properties

  • Property disposal: Methods for selling or losing properties

  • Mortgage handling: Methods for mortgaging and unmortgaging properties

  • Development management: Tracks houses and hotels on properties

  • Group completion checking: Identifies completed property sets

  • Development possibility checking: Determines if houses or hotels can be built

  • Property value calculation: Calculates total value of owned properties

Jail System

The Player class manages jail interactions:

  • Jail state tracking: Monitors if player is in jail and for how many turns

  • Jail cards management: Tracks and uses Get Out of Jail Free cards

  • Jail strategy: Handles decision-making for leaving jail

  • AI jail behavior: Special AI logic for jail situations

  • Forced release handling: Manages forced release after three turns

  • Stay in jail option: New strategic option to remain in jail

Exit and Bankruptcy Systems

The Player class handles player exits from the game:

  • Bankruptcy processing: When a player can’t pay a debt * Liquidates all assets * Transfers properties to creditor if applicable * Removes player from active game

  • Voluntary exit handling: When a player chooses to leave * Calculates and records final assets * Returns all properties to the bank * Visually indicates exited status * Maintains player in display but marks as inactive

AI Player Integration

The Player class has special handling for AI players:

  • AI controller association: Links to appropriate AI logic based on difficulty

  • Decision delegation: Forwards key decisions to AI controller

  • Visual differentiation: Distinct appearance for AI players

  • Automated actions: Handles AI-specific automated behaviors

  • Difficulty level support: Different behavior based on AI difficulty

  • Hard AI emotion system: Integration with the emotional state system

Common Operations

The Player class provides several key operations:

  • move(steps): Move the player a specific number of spaces

  • start_move(path): Start moving the player along a specific path of spaces

  • draw_player(screen, x, y): Render the player token at specified coordinates

  • update_animation(): Update animation state for the current frame

  • pay(amount): Remove money from the player

  • receive(amount): Add money to the player

  • buy_property(property): Purchase a property

  • handle_bankruptcy(creditor): Process bankruptcy, optionally transferring assets to a creditor

  • handle_voluntary_exit(): Process player voluntarily leaving the game

  • get_total_assets(): Calculate total value of player’s money and properties
