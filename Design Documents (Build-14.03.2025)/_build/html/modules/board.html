Board Module

This module contains the Board class that represents the game board and manages the visual representation and interaction with the game board.

Key Features

  • Board initialization and setup: Creates and configures the visual game board

  • Visual rendering: Renders the game board and all its elements

  • Camera controls: Advanced zooming and panning system

  • Player token positioning: Manages player token placement and animation

  • Property visualization: Shows property ownership and development

  • Message display system: Shows game events and notifications

  • Space management: Handles properties, special spaces, and their interactions

  • Animation system: Provides smooth animations for player movement

  • Interactive elements: Allows clicking on properties and spaces

  • Visual scaling: Adapts to different screen sizes and resolutions

  • Dynamic board generation: Creates board layouts based on window size

Board Structure

The game board consists of 40 spaces arranged in a square layout:

  • Corner spaces: GO, Jail, Free Parking, Go to Jail

  • Property spaces: Color-coded by property groups

  • Utility spaces: Electric Company, Water Works

  • Station spaces: Train stations

  • Special spaces: Pot Luck, Opportunity Knocks, Tax spaces

Camera Controls

The board implements a comprehensive camera system through the CameraControls class:

  • Zoom in/out: Using +/- keys to adjust zoom level

  • Pan the board: Using WASD or arrow keys to move the view

  • Zoom limits: Enforces minimum and maximum zoom levels

  • Speed adjustment: Camera move speed adjusts based on zoom level

  • Bound checking: Prevents panning too far from the board

  • Automatic positioning: Centers view on important game events

Player Animation System

The Board module features an advanced player animation system:

  • Movement animation: Smooth transitions between spaces

  • Path generation: Creates movement paths for dice roll results

  • Progress tracking: Tracks animation progress for each player

  • Multiple player positioning: Intelligently positions multiple players on the same space

  • Visual effects: Bounce effects, glowing, and highlighting

  • Corner handling: Special positioning logic for corner spaces

Message Display System

The board includes a game message display system:

  • Message queue: Maintains recent game messages

  • Scrolling display: Shows multiple messages in chronological order

  • Timed messages: Messages have display durations

  • Visual styling: Messages appear in a semi-transparent overlay panel

  • Automatic updates: New messages are added to the display in real-time

  • History tracking: Maintains a limited history of recent messages

CameraControls Class

The Board module includes a CameraControls class that manages the zooming and panning functionality:

  • Zoom level: Manages current zoom level (default: 1.0)

  • Offsets: Tracks current camera offset (x, y)

  • Move speed: Configurable movement speed

  • Zoom speed: Configurable zoom speed

  • Input handling: Processes keyboard input for camera movement

  • Limits enforcement: Maintains zoom and pan within acceptable ranges

Board Visualization

The board visualization includes several key components:

  • Base board image: Renders the main game board image

  • Property ownership indicators: Visual indicators of property ownership

  • Player token positioning: Shows player positions on the board

  • Houses and hotels display: Visualizes property development

  • Message notifications: Displays game events and important messages

  • Highlighting of active spaces: Visually identifies active or relevant spaces

  • Visual effects: Special effects for game events

  • Background elements: Renders a themed background behind the board

Space Interaction

The Board class enables interaction with board spaces:

  • Property clicking: Detects clicks on properties for development or info

  • Space identification: Maps screen coordinates to board positions

  • Space information: Provides details about spaces at specific positions

  • Position conversion: Converts between board and screen coordinates

  • Property grouping: Identifies properties belonging to the same group

  • Visual feedback: Provides visual feedback on interactive elements

Common Operations

The Board class provides several key operations:

  • draw(screen): Render the board and all its elements on the screen

  • update_board_positions(): Update the positions of all elements on the board

  • draw_player(screen, player, rect, index): Render a player token at a specific position

  • get_space(position): Get the space data at a specific position

  • update_ownership(properties_data): Update property ownership visualization

  • property_clicked(pos): Handle mouse clicks on properties

  • add_message(text): Add a message to the board’s message display

  • board_to_screen(x, y): Convert board coordinates to screen coordinates

  • update_offset(dx, dy): Update the camera offset

Technical Implementation

The Board class uses several techniques for efficient rendering:

  • Surface composition: Uses multiple surfaces for layered rendering

  • Alpha blending: Implements transparency for visual effects

  • Rect management: Efficiently handles rectangles for space positioning

  • Dynamic scaling: Adjusts elements based on screen size and zoom level

  • Surface caching: Optimizes rendering performance

  • Error handling: Gracefully handles missing assets or rendering errors

  • Coordinate transformation: Converts between different coordinate systems
