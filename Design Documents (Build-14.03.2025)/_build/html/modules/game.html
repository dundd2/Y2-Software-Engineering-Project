Game Module

This module contains the main Game class which orchestrates the game flow and manages the overall game state. It serves as the central controller for the Property Tycoon game, handling everything from game initialization to player turns, property management, and win conditions.

Key Features

  • Game initialization and setup: Creates and configures the game environment, board, and players

  • Turn management: Controls the flow of player turns and actions

  • Player actions handling: Processes player decisions like buying properties, paying rent, and development

  • Game state management: Tracks and updates the state of the game, including player positions and property ownership

  • Win condition checking: Determines when the game ends and identifies the winner

  • Save/Load game functionality: Allows games to be saved and resumed later

  • Multiple game modes: Supports different game modes including full game and time-limited games

  • AI player support: Includes AI players with different difficulty levels (Easy/Hard)

  • Visual rendering: Handles the graphical representation of the game state

  • User interface: Manages buttons, notifications, and interactive elements

  • Animation system: Provides smooth animations for player movement and dice rolling

  • Free Parking pot: Tracks and manages the Free Parking money pot

  • AI emotion system: Interfaces with the Hard AI’s emotional state system

Game Modes

The Game class supports multiple game modes:

  1. Full Game: The traditional game that continues until only one player remains solvent

  2. Abridged Game: A time-limited version that ends after a specified duration, with the winner determined by total assets

  3. Pause functionality: Allows players to pause the game during time-limited modes

User Interface Components

The Game class manages several UI components:

  • Board visualization: Renders the game board with property ownership indicators

  • Player information: Displays player stats including money, properties, and position

  • Action buttons: Provides interactive buttons for player decisions

  • Dice animation: Visualizes dice rolls with animated graphics

  • Property cards: Shows detailed information about properties

  • Notifications: Displays game events and important messages

  • Development UI: Interface for building houses and hotels

  • Auction system: UI for property auctions when players decline to purchase

  • Jail options: Interface for handling jail choices

  • Emotion UI: Interface for interacting with AI emotional states

  • Free Parking display: Shows the current amount in the Free Parking pot

  • Time remaining: Displays countdown timer in abridged mode

Game States

The Game class uses a state machine approach to manage different game phases:

  • ROLL: Waiting for the current player to roll dice

  • BUY: Asking the current player if they want to buy the property they landed on

  • AUCTION: Running an auction for a property that was declined for purchase

  • DEVELOPMENT: Managing property development (building houses/hotels)

Game Initialization

The Game class is initialized with several key parameters:

  • players: List of Player objects representing the game participants

  • game_mode: String indicating the game mode (“full” or “abridged”)

  • time_limit: Optional time limit in seconds for abridged mode

  • ai_difficulty: String indicating AI difficulty level (“easy” or “hard”)

The initialization process includes: * Loading game assets (dice images, background, etc.) * Creating the game board and configuring player tokens * Initializing the game logic * Setting up UI components * Creating appropriate AI players based on difficulty * Preparing card decks

Visual Elements

The Game class includes various visual elements:

  • Modern button design: Custom buttons with gradients, shadows, and hover effects

  • Property cards: Detailed cards showing property information

  • Property tooltips: Hover tooltips with key property details

  • Dice visualization: Animated dice with special effects for doubles

  • Notifications: Popup notifications for important events

  • Card display: Visual representation of drawn cards

  • Development interface: UI for property development

  • Player status panel: Shows each player’s current status and assets

  • Animation effects: Smooth player movement, dice rolling, etc.

Property Systems

The Game class provides comprehensive property management:

  • Property purchase: Interface for buying properties

  • Property auction: Full auction system with bidding interface

  • Property development: Building and selling houses and hotels

  • Mortgage management: Interface for mortgaging and unmortgaging properties

  • Property tooltip: Hover information about properties

  • Property card: Detailed property information display

  • Development notification: Alert when development is available

Free Parking System

The Game class implements a Free Parking money pot:

  • Visual display: Shows the current amount in the Free Parking pot

  • Collection mechanism: Players collect the pot when landing on Free Parking

  • Contribution sources: Money from fines, taxes, and certain cards go to the pot

  • Automatic update: Synchronizes with the game logic’s Free Parking fund

Jail System

The Game class provides a comprehensive jail interface:

  • Jail options UI: Interface for choosing how to handle being in jail

  • Visual indicator: Shows which players are currently in jail

  • Choice options: Pay fine, use card, roll for doubles, or stay in jail

  • AI jail handling: Automated jail decisions for AI players

  • Forced payment: Automatic handling of mandatory release after three turns

Auction System

The Game class implements a full auction system:

  • Auction UI: Interface for bidding on properties

  • Bidding mechanism: Players can enter bids or pass

  • AI bidding: Automated bidding for AI players

  • Visual feedback: Shows current highest bidder and minimum bid

  • Time limit: Auctions have a time limit for decisions

  • Result display: Shows auction winner and final price

Time Limit System (Abridged Mode)

The Game class includes a comprehensive time management system:

  • Timer display: Shows remaining time in abridged mode

  • Visual countdown: Progress bar indicates time remaining

  • Warning system: Visual and text warnings when time is running low

  • Final lap mechanism: When time expires, players complete their current lap

  • Pause functionality: Allows pausing the timer during the game

  • Asset calculation: Determines winner based on assets when time expires

AI Integration

The Game class integrates with AI players:

  • AI decision making: Automated handling of AI player turns

  • AI difficulty levels: Support for easy and hard AI players

  • Emotional system interface: Visual interface for Hard AI emotional states

  • AI taunt system: Players can influence AI mood through interaction

  • Automated development: AI can develop properties during their turn

  • Decision visualization: Shows AI decisions during their turn

Key Methods

  1. draw(): The main rendering method that draws the game state

  2. handle_click(), handle_motion(), handle_key(): Process user input

  3. play_turn(): Executes a player’s turn including dice roll and movement

  4. handle_space(): Processes the effect of landing on a space

  5. handle_buy_decision(): Manages property purchase decisions

  6. start_auction(): Initiates the property auction process

  7. handle_jail_turn(): Manages a player’s options when in jail

  8. show_notification(): Displays notification messages to the player

  9. handle_card_draw(): Processes drawing and resolving cards

  10. check_time_limit(): Checks if the time limit has been reached in abridged mode

  11. handle_voluntary_exit(): Processes a player choosing to leave the game

  12. check_game_over(): Determines if the game has ended and identifies the winner

  13. draw_development_ui(): Shows the property development interface

  14. update_ai_mood(): Updates the emotional state of Hard AI players

Game Event Handling

The Game class provides comprehensive event handling:

  • Animation control: Manages smooth animations between game states

  • Turn progression: Ensures proper flow between player turns

  • Card resolution: Handles the effects of drawn cards

  • Space landing effects: Processes landing on different board spaces

  • Win condition checking: Constantly monitors for game end conditions

  • Input handling: Processes mouse and keyboard input based on game state

  • AI turn automation: Automatically handles AI player turns

  • Time limit checking: Monitors time limit in abridged mode

  • Synchronization: Ensures game state is consistent between components
