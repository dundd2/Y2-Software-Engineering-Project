UI Module

This module contains the user interface components of the game, providing a modern, interactive interface for players to navigate menus, configure game settings, and interact with the game.

Key Features

  • Modern UI Components: Custom buttons, input fields, and interactive elements

  • Multiple Game Pages: Various screens for different game states and configurations

  • Responsive Design: Adapts to different screen resolutions

  • Visual Effects: Gradients, shadows, and animations for enhanced visual appeal

  • User Input Handling: Comprehensive mouse and keyboard interaction

  • Game Configuration: Settings for player count, AI difficulty, and game modes

  • Consistent Styling: Unified color scheme and design language

  • Token Selection: Customizable player tokens with visual selection interface

  • Emotional AI Interface: Visual feedback and interaction with AI emotional states

UI Components

The UI module implements several reusable components:

  1. ModernButton: Interactive buttons with hover effects, images, and selection states

  2. ModernInput: Text input fields for player names and other user input

  3. BasePage: Foundation class for all game pages with common functionality

  4. DevelopmentNotification: Notification system for property development

  5. AIEmotionUI: Interface to interact with and view AI emotional states

  6. TokenSelection: Interface for selecting and customizing player tokens

Color Scheme

The UI uses a consistent color palette:

  • Background: Dark modern background (18, 18, 18)

  • Accent Color: Burgundy highlight (128, 0, 32)

  • Button Hover: Lighter burgundy (160, 20, 40)

  • Text Colors: White, cream, and black for different contexts

  • Status Colors: Error red (220, 53, 69) and success green (0, 100, 0)

  • Player Colors: Green for human players, red for AI players

  • Gold Accents: For highlighting and borders (218, 165, 32)

Game Pages

The module implements various game pages, each with specialized functionality:

  1. MainMenuPage: * Main entry point with options to start game, view instructions, and access settings * Links to GitHub repository and YouTube content * Version display and keyboard shortcut hints

  2. SettingsPage: * Configure screen resolution with multiple preset options * Font selection with real-time preview * Settings confirmation and application

  3. StartPage: * Comprehensive player setup interface * Configure human and AI player counts (up to 5 total players) * Customizable player names and token selection * Visual feedback for valid configurations

  4. PlayerSelectPage: * Alternative player selection interface with simplified controls * Direct name input for multiple players * Navigation controls with keyboard shortcuts

  5. GameModePage: * Select between full game and abridged game modes * Configure time limits for abridged mode * Clear explanations of game mode differences

  6. EndGamePage: * Displays game results with winner announcement * Shows final assets for all players * Lists bankrupted players and voluntary exits * Displays laps completed statistics * Visual celebration effects (confetti animation) * Options to play again or quit

  7. HowToPlayPage: * Game instructions and rules explanation * Visual guides and examples * Quick access to return to the main menu

  8. AIDifficultyPage: * Select between Easy and Hard AI difficulty levels * Explanations of AI behavior differences * Visual distinction between difficulty options

Special UI Components

The module includes specialized components for enhancing gameplay:

  1. DevelopmentNotification: * Alerts players when they can develop properties * Visual notification with property development options * Interactive confirmation buttons

  2. AIEmotionUI: * Displays current AI emotional state * Interactive buttons to taunt or please the AI * Visual feedback showing AI mood changes * Easter egg feature for repeated interactions

  3. Token Selection Interface: * Visual selection of player tokens * Token swapping between players * Preview of selected tokens * Prevention of token conflicts

Input Handling

Each page implements three key methods for user interaction:

  • handle_click: Process mouse clicks on UI elements with appropriate responses

  • handle_motion: Track mouse movement for hover effects and visual feedback

  • handle_key: Process keyboard input for navigation, shortcuts, and text entry

Media Integration

The UI integrates various media elements:

  • Background Images: Custom backgrounds for different pages

  • Logo Display: Game logo with visual effects

  • Player Tokens: Customizable player token images

  • Emotional Icons: Happy and angry faces for AI interaction

  • External Links: Integration with GitHub repository and external content

Font Management

The UI provides font customization:

  • Multiple Font Options: Different font styles available

  • Dynamic Font Sizing: Automatic scaling based on screen resolution

  • Text Scaler Integration: Works with the text_scaler module for consistent sizing

  • Font Preview: Real-time preview of font changes in settings

BasePage Class

The BasePage class serves as the foundation for all game pages with the following features:

  • Background Rendering: Draws the game background with gradients or custom images

  • Title Display: Shows the game logo with visual effects

  • Font Management: Initializes and manages different font sizes

  • Instructions Display: Shows help text and instructions

  • Asset Loading: Handles loading and scaling of image assets

ModernButton Class

The ModernButton class implements interactive buttons with:

  • Visual States: Normal, hover, active, and selected states

  • Custom Styling: Configurable colors and appearance

  • Image Support: Optional button images

  • Interaction Handling: Methods to check for mouse hover and clicks

  • Visual Effects: Gradient, shadow, and highlight effects

  • Border Styling: Custom border radius and colors

ModernInput Class

The ModernInput class provides text input fields with:

  • Text Entry: Support for keyboard text input

  • Visual Feedback: Active, inactive, and error states

  • Placeholder Text: Default text when empty

  • Selection State: Visual indication when selected

  • Background Transparency: Configurable transparency level

  • Error Highlighting: Visual indication of invalid input

AIEmotionUI Class

The AIEmotionUI class provides an interface for interacting with AI emotions:

  • Emotional State Display: Shows current AI mood value

  • Interactive Buttons: Allows players to make AI happier or angrier

  • Visual Feedback: Shows effects of player interactions

  • Mood Tracking: Displays numerical value of AI mood

  • Easter Egg: Hidden feature activated by excessive interaction

  • Integration: Works with Hard AI’s emotional system

Page Navigation and State Management

The UI system implements a sophisticated state management approach:

  1. Pages maintain their own state between interactions

  2. Player preferences and settings are preserved across page transitions

  3. Game configuration is built step by step through multiple pages

  4. Each page validates input before allowing progression

  5. Pages communicate results through standardized return values
