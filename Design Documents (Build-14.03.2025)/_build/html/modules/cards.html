Cards Module

This module contains classes and functions related to the game cards, including the Card and CardDeck classes that manage the Pot Luck and Opportunity Knocks cards in the game.

Key Features

  • Card Types Management: Handles two distinct card types - Pot Luck and Opportunity Knocks

  • Card Deck Initialization: Automatically loads and shuffles card data from game_logic module

  • Card Drawing: Provides mechanisms for drawing cards from decks

  • Discard Pile Management: Tracks drawn cards and reshuffles when necessary

  • Special Card Handling: Special handling for “Get Out of Jail Free” cards

  • Card Actions: Each card contains text and an associated action function with game impact

  • Deck State Tracking: Monitors the number of cards in each deck and discard pile

  • Interactive Cards: Support for cards that require player decisions

Card Types

The game features two types of cards:

  1. Pot Luck Cards: Similar to Chance cards in traditional Monopoly, these cards can provide benefits or penalties to players. They include actions like collecting money, paying fines, or moving to specific board locations.

  2. Opportunity Knocks Cards: Similar to Community Chest cards in traditional Monopoly, these cards also provide various effects that can help or hinder players.

Card Structure

Each card contains:

  • Text: The message displayed to the player when the card is drawn

  • Action: A function that implements the card’s effect on the game state

  • Card Type: Identifies whether the card is a Pot Luck or Opportunity Knocks card

  • Requires Input Flag: Indicates if the card requires player input (e.g., choice between paying a fee or drawing another card)

  • Special Card Flag: Indicates if the card has special effects (like jail free, movement, etc.)

Card Effects

Cards can have various effects on gameplay:

  • Money Transfers: Adding or removing money from a player

  • Movement: Moving a player to a specific location on the board

  • Property Interactions: Charging players based on their property developments

  • Special Actions: Providing benefits like “Get Out of Jail Free”

  • Player-to-Player Interactions: Cards like birthday collection that involve multiple players

  • Decision Points: Cards that offer choices to players

  • Repair Assessments: Charges based on property development levels

CardType Enum

The CardType enum defines the two types of cards in the game:

  • POT_LUCK: Represents Pot Luck cards

  • OPPORTUNITY_KNOCKS: Represents Opportunity Knocks cards

Card Class

The Card class represents an individual card in the game with the following attributes:

  • text: The message displayed when the card is drawn

  • action: The function that implements the card’s effect

  • card_type: The type of card (Pot Luck or Opportunity Knocks)

  • requires_input: Flag indicating if the card requires player input

  • is_special: Flag indicating if the card has special effects like jail free, movement, etc.

CardDeck Class

The CardDeck class manages a deck of cards with the following attributes and methods:

  • card_type: The type of cards in this deck

  • cards: The list of cards currently in the deck

  • discard_pile: The list of cards that have been drawn and discarded

  • last_drawn_card: Reference to the most recently drawn card

  • initialize_deck(): Sets up the card deck from data in game_logic module

  • draw_card(): Takes the top card from the deck

  • return_card(card, to_bottom): Places a card in the discard pile

  • return_jail_card(card_type): Returns a Get Out of Jail Free card to the deck

  • peek_top_card(): Looks at the top card without drawing it

  • get_remaining_count(): Returns the number of cards left in the deck

  • get_discard_count(): Returns the number of cards in the discard pile

Card Implementation

The card data is defined in the game_logic.py module and imported into the Cards module. Each card includes:

  • Text for display to the player

  • An action function that implements the card’s game effect

  • The action functions use lambda expressions to provide different effects

The card actions can refer to: * player: The current player who drew the card * bank: The bank’s current money amount * free_parking: The free parking fund amount * board: Optional reference to the board object for more complex operations

Card Action Examples

Examples of card actions include:

```python # Simple money transfer {“text”: “You inherit £200”,

“action”: lambda player, bank, free_parking: (player[‘money’] + 200, bank - 200, free_parking)}

# Movement to specific location {“text”: “Advance to GO”,

“action”: lambda player, bank, free_parking: (1, bank, free_parking)}

# Payment with Free Parking option {“text”: “Pay a £10 fine or take opportunity knocks”,

“action”: lambda player, bank, free_parking, board=None: board.handle_payment_to_bank(player, 10, True)}

# Property repair assessment {“text”: “You are assessed for repairs, £40/house, £115/hotel”,

“action”: lambda player, bank, free_parking, board=None: board.handle_repair_assessment(player, 40, 115)}

# Player-to-player interaction {“text”: “It’s your birthday. Collect £10 from each player”,

“action”: lambda player, bank, free_parking, board=None: board.handle_birthday_collection(player)}

```

Integration with Game Logic

The Cards module integrates closely with the Game Logic module through:

  • Importing card definitions from game_logic.py

  • Action functions that reference board methods for complex operations

  • Card effects that can modify player state, bank funds, and free parking

  • Special handling for cards that require player input or decisions

This design allows the card system to be flexible while maintaining clean separation of concerns between modules.
