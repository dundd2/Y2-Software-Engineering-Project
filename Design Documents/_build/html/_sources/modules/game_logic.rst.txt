Game Logic Module
=================

This module contains the game logic functions and classes that drive the game mechanics, including the core GameLogic class that manages the rules and flow of the Property Tycoon game.

Key Components
--------------

* **Core Game Rules**: Implementation of Property Tycoon game rules and mechanics
* **Turn Management**: Control of player turns and game progression
* **Property Transactions**: Handling property purchases, rents, and auctions
* **Development Mechanics**: Management of house and hotel building
* **Chance Cards**: Implementation of "Opportunity Knocks" and "Pot Luck" card effects
* **Money Management**: Tracking of player finances and bank transactions
* **AI Integration**: Interface with AI player decision-making systems

Game Flow
---------

The game follows a structured flow, managed by the GameLogic class:

.. plantuml::

   @startuml
   
   title Property Tycoon Game Flow
   
   start
   
   :Game Initialization;
   note right
     - Load property data
     - Create players
     - Set up cards
     - Initialize bank
   end note
   
   repeat
     :Start Player Turn;
     
     if (Player in Jail?) then (Yes)
       :Handle Jail Turn;
       if (Remains in Jail?) then (Yes)
         :End Turn;
       else (No)
         :Allow Normal Turn;
       endif
     else (No)
       :Roll Dice;
       
       if (Rolled Doubles?) then (Yes)
         :Increment Doubles Count;
         if (Doubles Count = 3?) then (Yes)
           :Send to Jail;
           :End Turn;
         endif
       else (No)
         :Reset Doubles Count;
       endif
       
       :Move Player;
       
       if (Landed on Property?) then (Yes)
         if (Property Owned?) then (Yes)
           if (Owner is Current Player?) then (Yes)
             :No Action;
           else (No)
             :Pay Rent;
           endif
         else (No)
           :Property Purchase Decision;
           if (Buy?) then (Yes)
             :Process Purchase;
           else (No)
             :Auction Property;
           endif
         endif
       else (No)
         if (Special Square?) then (Yes)
           :Process Special Square Effects;
         endif
       endif
       
       :Development Phase;
       
       :Trading Phase;
     endif
     
     :Check for Bankruptcy;
     if (Player Bankrupt?) then (Yes)
       :Remove from Game;
     endif
     
     :Move to Next Player;
   repeat while (Game End Condition Not Met)
   
   :Determine Winner;
   
   stop
   
   @enduml

Special Square Handling
-----------------------

The GameLogic module includes specialized handling for different square types:

* **Go**: Players collect money when passing or landing on Go
* **Jail**: Players must decide how to get out of jail (roll doubles, pay fine, or use card)
* **Free Parking**: Players may collect accumulated funds (depending on game settings)
* **Go to Jail**: Players are immediately sent to jail
* **Tax Squares**: Players must pay a tax to the bank
* **Opportunity Knocks/Pot Luck**: Players draw a card and follow its instructions

Trade System
------------

The GameLogic module includes a trading system for players to exchange properties and money:

* **Property Exchange**: Players can trade properties with each other
* **Cash Adjustment**: Money can be included in trades to balance the exchange
* **Trade Negotiation**: Players can propose, accept, or reject trades
* **AI Trade Evaluation**: AI players evaluate trades based on property values and strategic considerations

AI Player Integration
---------------------

The GameLogic class integrates with AI player logic:

* **AI Decision Making**: Handles AI player decisions during their turn
* **Property Purchase Decisions**: AI evaluates whether to buy properties
* **Auction Bidding**: AI determines appropriate bid amounts
* **Bankruptcy Prevention**: AI strategizes to avoid bankruptcy
* **Property Development**: AI decides when to build houses/hotels
* **Mortgage Management**: AI decides when to mortgage or unmortgage properties

Game State Management
---------------------

The module maintains the complete game state, tracking:

* **Player Positions**: Current location of each player on the board
* **Property Ownership**: Which player owns each property
* **Development Level**: Number of houses or hotels on each property
* **Financial State**: Amount of money held by each player and the bank
* **Special Card Ownership**: Which players hold "Get Out of Jail Free" cards
* **Jail Status**: Which players are in jail and for how many turns
* **Bankruptcy Status**: Which players have gone bankrupt
* **Game Messages**: Queue of game events to display to players
