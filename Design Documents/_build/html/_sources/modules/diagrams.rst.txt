=======================

This section provides visual representations of the game's flow and structure using diagrams.

Main Game Flow
--------------

The main game flow illustrates the sequence of play in Property Tycoon:

.. plantuml::

   @startuml
   
   title Property Tycoon Game Flow
   
   start
   
   :Game Initialization;
   note right
     Load assets, create players,
     initialize board and properties
   end note
   
   :Player Setup;
   note right
     Select number of players
     Choose tokens
     Configure AI players
   end note
   
   :Game Start;
   
   while (Game Not Over?) is (Yes)
     :Begin Player Turn;
     
     if (Player in Jail?) then (Yes)
       :Handle Jail Turn;
       if (Can Exit Jail?) then (Yes)
         :Exit Jail;
       endif
     else (No)
       :Roll Dice;
       
       if (Rolled Doubles 3 Times?) then (Yes)
         :Go to Jail;
       else (No)
         :Move Player;
         
         :Handle Landing Square;
         note right
           Process effects based
           on the square type
         end note
         
         if (Square Type?) then (Property)
           if (Property Owned?) then (Yes)
             :Pay Rent;
           else (No)
             :Offer to Buy Property;
             if (Buy Property?) then (Yes)
               :Purchase Property;
             else (No)
               :Auction Property;
             endif
           endif
         else if (Square Type?) then (Card)
           :Draw Card;
           :Apply Card Effect;
         else if (Square Type?) then (Go To Jail)
           :Send to Jail;
         else if (Square Type?) then (Tax)
           :Pay Tax;
         else (Other)
           :Handle Special Square;
         endif
         
         :Property Development Phase;
         note right
           Player can build houses,
           hotels, or mortgage properties
         end note
         
         if (Rolled Doubles?) then (Yes)
           :Extra Turn;
           backward:Begin Player Turn;
         endif
       endif
     endif
     
     :Check Player Bankruptcy;
     if (Player Bankrupt?) then (Yes)
       :Process Bankruptcy;
     endif
     
     :End Turn;
     
     :Check Game End Condition;
     if (End Condition Met?) then (Yes)
       :Game Over;
       break
     endif
     
     :Next Player;
   endwhile
   
   :Determine Winner;
   
   :Show Game Results;
   
   stop
   
   @enduml

Hard AI Emotional System
------------------------

The Hard AI uses an emotional system to make its decisions more human-like and unpredictable:

.. plantuml::

   @startuml
   
   title Hard AI Emotional System
   
   state "Neutral" as Neutral
   state "Happy" as Happy
   state "Strategic" as Strategic
   state "Angry" as Angry
   state "Desperate" as Desperate
   state "Vengeful" as Vengeful
   
   [*] --> Neutral
   
   Neutral --> Happy : Positive Events\nGood Fortune
   Neutral --> Strategic : Acquiring Properties\nGaining Monopoly
   Neutral --> Angry : Negative Events\nPaying High Rent
   
   Happy --> Neutral : Time Passes\nMild Setbacks
   Happy --> Strategic : Opportunity Arises
   
   Strategic --> Neutral : Plans Disrupted
   Strategic --> Happy : Plan Succeeds
   Strategic --> Vengeful : Direct Competition
   
   Angry --> Neutral : Time Passes\nPositive Events
   Angry --> Desperate : Financial Trouble
   Angry --> Vengeful : Player Causes Problems
   
   Desperate --> Neutral : Financial Recovery
   Desperate --> Angry : Further Setbacks
   
   Vengeful --> Angry : Target Unavailable
   Vengeful --> Neutral : Revenge Satisfied\nTime Passes
   
   note right of Happy
     Risk tolerance increases
     Values properties higher
     More likely to develop
   end note
   
   note right of Strategic
     Focuses on monopolies
     Saves money strategically
     Calculated development
   end note
   
   note right of Angry
     Less rational decisions
     More aggressive bidding
     Less strategic planning
   end note
   
   note right of Desperate
     Sells/mortgages assets
     Takes high risks
     Focuses on survival
   end note
   
   note right of Vengeful
     Targets specific player
     Makes spite-based decisions
     May overpay to block
   end note
   
   @enduml

Game State Machine
------------------

The game uses a state machine to manage different phases of gameplay:

.. plantuml::

   @startuml
   
   title Game State Machine
   
   [*] --> MENU
   
   state MENU {
     [*] --> MAIN_MENU
     MAIN_MENU --> PLAYER_SETUP : Start Game
     PLAYER_SETUP --> RULE_SELECTION : Next
     RULE_SELECTION --> GAME_SETUP : Start
     MAIN_MENU --> OPTIONS : Options
     OPTIONS --> MAIN_MENU : Back
     MAIN_MENU --> HELP : Help
     HELP --> MAIN_MENU : Back
}}}}}}}}}}}}}}}}}}}}}}}}}
   
   MENU --> PLAYING : Game Start
   
   state PLAYING {
     [*] --> WAITING_FOR_ROLL
     
     WAITING_FOR_ROLL --> MOVING : Roll Dice
     MOVING --> PROPERTY_DECISION : Land on Unowned Property
     MOVING --> PAY_RENT : Land on Owned Property
     MOVING --> DRAW_CARD : Land on Card Space
     MOVING --> SPECIAL_SQUARE : Land on Special Square
     
     PROPERTY_DECISION --> WAITING_FOR_NEXT_PLAYER : Buy/Pass
     PAY_RENT --> WAITING_FOR_NEXT_PLAYER
     DRAW_CARD --> CARD_EFFECT
     CARD_EFFECT --> WAITING_FOR_NEXT_PLAYER
     SPECIAL_SQUARE --> SPECIAL_EFFECT
     SPECIAL_EFFECT --> WAITING_FOR_NEXT_PLAYER
     
     WAITING_FOR_NEXT_PLAYER --> WAITING_FOR_ROLL : Next Player
}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}}
   
   PLAYING --> DEVELOPMENT_MODE : Development Button
   DEVELOPMENT_MODE --> PLAYING : Exit Development
   
   PLAYING --> AUCTION_MODE : Property Auctioned
   AUCTION_MODE --> PLAYING : Auction Complete
   
   PLAYING --> TRADE_MODE : Trade Button
   TRADE_MODE --> PLAYING : Trade Complete
   
   PLAYING --> GAME_OVER : End Condition Met
   
   GAME_OVER --> MENU : New Game
   
   @enduml
